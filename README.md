## 📋 Project Overview
This repository contains a full end-to-end MLOps pipeline designed to analyze high-resolution H&E-stained tissue patches. The goal is to act as a "second pair of eyes" for pathologists, automatically flagging regions of interest that exhibit high nuclear density and malignant cell morphology.

### Key Achievements
- **Dataset:** Handled over 300,000 real-world microscopic tissue images from the [PatchCamelyon (PCam)](https://patchcamelyon.grand-challenge.org/) benchmark.
- **Performance:** Achieved an **ROC-AUC of 0.96+** and **89% Accuracy** on a held-out validation set of 32,000 patches.
- **Explainability (XAI):** Implemented Gradient-weighted Class Activation Maps (Grad-CAM) to visualize the specific cellular structures driving the AI's predictions.

## 🏗️ Architecture & Tech Stack

### 1. The Deep Learning Engine (`src/`)
- **PyTorch & Torchvision:** Core framework for tensor manipulation and GPU acceleration.
- **EfficientNetB0:** Used transfer learning from ImageNet, fine-tuning the convolutional blocks specifically for histopathology.
- **HDF5:** Engineered custom `Dataset` and `DataLoader` classes to stream gigabytes of compressed `.h5` binary data efficiently without crashing system memory.
- **Data Augmentation:** Implemented stain normalization, random flips, and color jittering to ensure the model generalizes across different hospital scanners.

### 2. MLOps & Verification Gate (`.git/hooks/`)
- **TensorBoard:** Integrated real-time training metric tracking (Loss, Accuracy, AUC).
- **Automated Checkpointing:** Model weights (`best_model.pth`) are dynamically saved only when validation AUC improves.
- **Pre-Push Quality Gate:** A custom Git hook (`verify_run.py`) guarantees that code cannot be pushed to GitHub unless the model successfully meets a minimum performance threshold (AUC > 0.80) and strict linting (flake8) passes.

### 3. Cloud Training Migration (`notebooks/`)
- Migrated local CPU-bound execution to Google Colab.
- Consolidated the data-fetching and PyTorch training loops into `AI_Cancer_Detection_Colab.ipynb` to leverage a free NVIDIA T4 GPU, reducing training time from hours to minutes.

### 4. Clinical Web Dashboard (`web/`)
- A dark-mode, medical-grade interactive UI built with pure HTML, CSS, and Vanilla JavaScript.
- Fetches the dynamic `results/training_summary.json` generated by the PyTorch evaluation script.
- Uses **Chart.js** to dynamically render the Training History, ROC Curve, and Confusion Matrix.
- Includes a "Tissue Sample Explorer" showcasing real extracted malignant and normal samples from the dataset.

## 🚀 How to Run Locally

### 1. Setup Environment
```bash
python -m venv venv
source venv/bin/activate  # On Windows use: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Train the Model (Colab Recommended)
Because the dataset is 8GB+, training locally on a CPU is extremely slow. 
Upload the provided `notebooks/AI_Cancer_Detection_Colab.ipynb` to [Google Colab](https://colab.research.google.com/), select a **T4 GPU**, and hit Run. 
Once finished, download the resulting `model_and_results.zip` and extract its contents into the root of this repository.

### 3. View the Dashboard
Run a local web server to view the results:
```bash
cd web
python -m http.server 8000
```
Then navigate to `http://localhost:8000/` in your browser.
